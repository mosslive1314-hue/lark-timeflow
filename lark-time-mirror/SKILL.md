# 🪞 time-mirror v1.0.0 - 时间镜子

> **一面自动化的时间镜子，帮你看清 24 小时的真实分配。不判断好坏，不当监工，不制造焦虑。**

**核心理念**:
> 一天只有 24 小时。  
> 8 小时睡眠，8 小时生活，8 小时学习工作。  
> 时间用在哪里，自然就会收回什么。

time-mirror 不会告诉你「该做什么」，它只会告诉你「你做了什么」。  
**它是一面镜子，不是一个监工。**

## 🎯 定位

自动追踪 macOS App 使用时间 → 整合分析 → 写入 **Apple 日历 + 飞书多维表格** → 每晚提醒补全空白 → 每周自动生成复盘统计。

**适合人群**:
- 想知道自己的时间真正去哪了
- 讨厌手动记录时间，觉得太麻烦
- 已经在用飞书，想要飞书内的提醒和看板
- 相信「看清事实」比「强行自律」更重要

## ✨ 功能

| 功能 | 状态 | 说明 |
|------|------|------|
| 自动采集 macOS App 使用时间 | ✅ | 利用系统内置 knowledgeC 数据库，不需要额外安装监控 |
| 采集 Chrome 浏览历史 | ✅ | 结合 URL 智能分类 |
| 整合聚合成 30 分钟粒度时间块 | ✅ | 合并相邻同类活动 |
| 识别空白时段（> 30 分钟无活动） | ✅ | 提醒用户补全 |
| 写入 Apple 日历 | ✅ | iCloud 同步到所有设备 |
| 写入飞书多维表格 | ✅ | 支持统计分析和可视化看板 |
| **飞书机器人每晚提醒补全** | ✅ 新增 | 自动发送当日时间线，@ 用户补全空白 |
| **飞书每周日自动复盘** | ✅ 新增 | 生成周统计报告，发布到飞书群 |
| **创建日程提醒** | ✅ 新增 | 支持基于时间块创建待办/提醒 |

## 🗺️ 工作流程

```

  系统自动记录
    ↓
  time-mirror 采集聚合
    ↓
  写入 Apple 日历 + 飞书多维表格
    ↓
  **飞书机器人** 每晚提醒 → 用户回复补全空白
    ↓
  每周日 **飞书机器人** 自动发布周复盘统计
    ↓
  你看清时间去向 → 做出调整 → 收获改变
```

## 🛠️ 飞书能力整合

### 1. 飞书多维表格
- 存储每日时间块数据
- 支持按日期/分类筛选
- 可自建仪表盘 → 饼图/柱状图展示每周时间分配
- 趋势分析 → 对比每周变化

### 2. 飞书机器人提醒
- **每晚 21:30** 自动发送当日时间线到飞书群
- 标注空白时段 ➡️ @你 补全
- 你在飞书回复内容 → 自动更新日历和多维表格

### 3. 飞书每周复盘
- **每周日晚 20:00** 自动生成周统计
- 各分类时间占比 + 对比上周变化
- 直接发送到飞书群，一目了然

### 4. 飞书日程创建
- 识别时间块中的待办 ➡️ 自动创建飞书日程
- 支持提醒通知 ➡️ 不错过重要事项

## 📐 飞书多维表格配置

### 表格结构
需要你在飞书多维表格创建以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 日期 | 日期 | 活动日期 |
| 时段 | 文本 | `08:00-08:22` |
| 分类 | 单选 | `💰工作 / 👀沟通 / 🛜产品化 / 🌳学习 / 🥬生存 / 🎮娱乐 / ❓空白` |
| 活动描述 | 文本 | `Figma 设计（穿插飞书/微信）` |
| 时长 | 数字 | 分钟 |
| 数据来源 | 单选 | `🤖自动采集 / ✏️手动补全` |

### 获取表格 ID
创建后获取 `appToken` 和 `tableId`，配置到环境变量：

```bash
export FEISHU_APP_TOKEN=your_app_token
export FEISHU_TABLE_ID=your_table_id
export FEISHU_WEBHOOK=your_robot_webhook
```

## 🧩 分类配置

编辑 `references/app_categories.md` 配置你的 App → 分类映射：

```markdown
# App → 分类映射
# 格式: App名称 = 分类 emoji 分类名称
# 示例:

Figma = 💰 工作
Google Chrome = 🌳 学习
"Google Chrome" = 💰 工作
微信 = 👀 沟通
飞书 = 👀 沟通
```

- App 名称和 macOS 显示一致
- 支持多个 App 映射到同一个分类
- Chrome 会根据 URL 智能推断分类

## 🚀 使用方式

### 每日采集（手动）

```bash
# 采集今天
./scripts/collect.sh

# 采集指定日期
./scripts/collect.sh 2026-04-03

# 采集后自动写入飞书多维表格
./scripts/collect-and-upload.sh 2026-04-03
```

### 自动定时（推荐）

在 crontab 中添加：

```cron
# 每晚 21:00 采集今天数据 + 写入飞书 + 发送提醒
0 21 * * * * cd /path/to/time-mirror && ./scripts/collect-and-upload.sh
```

统计周报会在采集时自动检测，如果是周日就发送周报。

## 🔒 安全隐私

- **所有数据仅在你自己的 Mac 和 你的飞书多维表格中**
- 不上传到任何第三方服务器
- 不对外分享你的个人数据
- 完全本地控制

## 🎯 设计哲学 - 少即是多

> 大多数时间管理工具的结局：第一周新鲜 → 第二周漏记 → 第三周放弃。

time-mirror 坚持**零摩擦**原则：

1. ✅ **不需要你手动记录** — 数据系统已经帮你记好了
2. ✅ **不需要你打开任何 App** — 飞书机器人直接找你
3. ✅ **不会打断你的心流** — 只在晚上问一次空白
4. ✅ **不判断对错** — 刷抖音 3 小时？记下来就好，你自己决定要不要改

> **"这个 skill 必须是工具，不能是主人。"**

## 🔒 安全边界

### 禁止操作
- ❌ 不上传数据到第三方服务器
- ❌ 不修改用户系统配置除了你手动添加的 crontab
- ❌ 不存储你的飞书凭证，只使用环境变量

### 需要确认
- ⚠️ 配置飞书 webhook 需要你提供 token，确认你理解数据安全风险
- ⚠️ 开启定时采集需要你手动修改 crontab
