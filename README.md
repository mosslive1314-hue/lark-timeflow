# 🪞 lark-timeflow - 飞书时间流

**时间追踪 ✦ 活动提醒 ✦ 智能待办 ✦ 日程管理**

> 你的飞书个人时间助理。看清时间去向，不错过重要活动，自动抓取待办日程。全平台适配，基座设计保持简洁。

```
┌─────────────────────────────────────────────────────────┐
│                    lark-timeflow                         │
│                                                         │
│  🪞 自动时间追踪    🔔 活动自动提醒    📝 智能待办抓取   │
│  📅 多日历支持     📊 每周自动复盘   🧱 基座可扩展      │
│  系统自动采集  →  飞书多维表格  →  机器人主动提醒       │
│  扫描群聊找活动/待办  →  确认自动加入日历  →  准时提醒   │
└─────────────────────────────────────────────────────────┘
```

<br>

## ✨ 核心理念

```
一天只有 24 小时。
8 小时睡眠，8 小时生活，8 小时学习工作。
时间用在哪里，自然就会收获什么。
```

**lark-timeflow 不会告诉你「该做什么」**，它只会：
- 告诉你**实际上你做了什么** → 时间镜子
- 告诉你**有什么新活动** → 活动闹钟
- 帮你**抓取讨论确定的待办** → 不用手动再记
- 帮你**管理日程** → 整合在飞书

**它是你的镜子，也是你的闹钟，不是你的监工。**

## 🖥️ 工作原理

### 完整数据流

```
┌──────────────┐     ┌──────────────┐     ┌────────────────┐
│  系统 (macOS/Win) │ │  lark-timeflow │ │  飞书         │
│  使用记录数据库   │─▶ │  采集+聚合   │─▶ │  多维表格     │
│  Chrome 历史      │   │  识别分类   │   │              │
├──────────────┤     ├────────────┤     ├────────────────┤
│  飞书群聊/私聊     │─▶ │ 扫描活动/待办 │─▶ │ 机器人问你     │
│             │     │ 提取信息   │     │ 确认添加提醒     │
├──────────────┤     ├────────────┤     ├────────────────┤
│             │     │  周日复盘     │─▶ │  发送周报       │
│             │     │  统计分析   │     │                │
└──────────────┘     └────────────┘     └────────────────┘
```

### 系统怎么知道你在用什么？

| 平台 | 方式 | 说明 |
|------|------|------|
| **macOS** | 自动读取 | 系统内置 `Knowledge 框架` 已经帮你记录好了 |
| **Windows** | 自动读取 | 读取系统自带「屏幕使用时间」数据库 |
| **Linux** | 导入 | 可配合 `activitywatch` 等工具导入 |
| **iPhone/iOS** | 截图识别 | 系统不开放数据库，你把「屏幕使用时间」截图发飞书 → AI 自动识别录入 |
| **Android** | 截图识别 | 把「数字健康/屏幕使用时间」截图发飞书 → AI 自动识别录入 |

> 💡 **说明**: 手机系统由于沙箱安全限制，第三方应用无法直接读取屏幕使用时间数据库，所以做不到全自动采集。但截图识别非常方便，截一张图发过来就搞定了，体验其实不差。

**lark-timeflow 只是读取/识别系统已有记录，不需要安装任何监控软件。**

## 🚀 快速开始

### 1. 配置环境变量

```bash
# 飞书配置
export FEISHU_APP_TOKEN=your_bitable_app_token
export FEISHU_TIME_TABLE_ID=your_time_daily_table_id
export FEISHU_EVENT_TABLE_ID=your_events_table_id
export FEISHU_WEBHOOK=your_robot_webhook
export FEISHU_CALENDAR_ID=your_feishu_calendar_id  # 可选，不填用默认
export DEFAULT_CALENDAR_TYPE=feishu  # feishu/apple/google
```

### 2. 添加定时任务

编辑 crontab: `crontab -e`

添加：

```cron
# 每晚 21:00 采集今日数据，发送提醒
0 21 * * * cd /path/to/lark-timeflow && ./scripts/collect-and-report.sh

# 每小时扫描一次群聊找新活动/待办
0 * * * * cd /path/to/lark-timeflow && ./scripts/fetch-events.sh
```

完成！**从此以后全自动运行**，不用你动手。

## 🗓️ 日历整合

lark-timeflow 会同时写入：
- **Apple 日历** → 自动通过 iCloud 同步到 iPhone
- **飞书日历** → 飞书手机端随时看，自动提醒

### App 分类映射

你可以在 `references/app-categories.md` 自定义配置：

```markdown
# App → 分类 配置
# 格式: App名称 = emoji 分类名称
Figma = 💰 工作
微信 = 👀 沟通
飞书 = 👀 沟通
Chrome = 🌳 学习
```

Chrome 会智能判断：根据访问URL 自动分类，比如 GitHub → 工作，抖音 → 娱乐。

<br>

## 📊 飞书多维表格 + 仪表盘

lark-timeflow 将所有时间块写入飞书多维表格，你可以自建仪表盘：

- 📊 今日时间分配饼图
- 📈 每周趋势柱状图
- 📅 月度汇总统计

**对比 Apple 日历**: Apple 日历适合看时间线，飞书多维表格适合看统计。两者互补。

<br>

## 📱 手机数据

目前**仅支持 macOS 自动采集**，iPhone 屏幕使用时间受沙箱限制没法自动采集。

替代方案：
- 你可以把 iPhone 屏幕使用时间截图发给飞书机器人，AI 自动识别录入
- 等待 OpenClaw iOS 端正式发布后支持自动采集

## ⚠️ 注意事项

| 问题 | 说明 |
|------|------|
| iCloud 同步延迟 | 远程模式下 iCloud 文件同步需要几分钟，脚本会自动等 |
| Chrome 锁定 | Chrome 运行时 History 文件锁定，复制的数据可能是几分钟前的，不影响使用 |
| 隐私 | **所有数据只在你的 Mac 和你的飞书，不上传第三方** |

## 🧘 设计哲学 - 少即是多

> 大多数时间管理工具的结局：第一周新鲜 → 第二周漏记 → 第三周放弃。

lark-timeflow 坚持**零摩擦**原则：

1. ✅ **不需要你手动记** → 系统已经帮你记好了，自动采集
2. ✅ **不需要你打开 App** → 机器人主动找你，不用你找它
3. ✅ **不会打断心流** → 只在晚上问一次空白，白天不打扰
4. ✅ **不判断对错** → 刷三小时抖音怎么了？记下来就好，改不改你决定

> *"工具就是工具，主人做主。"*
