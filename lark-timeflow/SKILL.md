# lark-timeflow v2.1.0 - 飞书时间流

> **你的飞书个人时间助理。看清时间去向，不错过重要活动，自动抓取待办日程。全平台适配，基座设计保持简洁。**

## 🎯 核心能力

| 能力 | 说明 |
|------|------|
| 🪞 **自动时间追踪** | 读取系统已有数据，自动统计每日时间分配，写入飞书多维表格。支持 macOS/Windows/Linux |
| 📊 **飞书仪表盘** | 多维表格自动统计，支持饼图/柱状图/周趋势 |
| 🔔 **活动自动提醒** | 抓取飞书群聊中的活动信息，报名后自动设置提醒 |
| 📝 **智能待办抓取** | 自动识别群聊/私聊中讨论确定的待办，主动询问是否添加日程 |
| 📅 **多日历支持** | 飞书日历 ✓ Apple 日历 ✓ Google Calendar ✓ 可配置 |
| 🤖 **飞书机器人交互** | 直接在飞书对话查询：「今天时间花哪了」「本周总结」「明天什么安排」 |
| 🧱 **基座设计** | 保持最小核心，可与其他技能配合扩展更多能力 |
| 🧘 **设计哲学** | 零摩擦不打断，镜子不是监工，提醒不搞骚扰 |

## 🗺️ 工作流

```
  每天自动采集
    ↓
  写入飞书多维表格 + Apple 日历
    ↓
  🤖 飞书机器人每晚提醒 → 你回复补全空白
    ↓
  每周日自动发周总结 → 你调整下周计划
    ↓
  抓取飞书群新活动 → 问你要不要报名 → 报名自动添加提醒
    ↓
  活动前准时提醒 → 不错过
```

## 🪞 时间追踪能力

- ✅ 智能 Chrome 分类（结合 URL 判断工作/学习/娱乐）
- ✅ 自动识别空白时段，飞书机器人提醒你补全

## 🔔 活动提醒 + 智能待办抓取

### 1. 活动提醒能力

**定时扫描你加入的飞书群聊**：
- 自动识别包含「活动」「报名」「讲座」「沙龙」「分享」「线下」关键词的消息
- 提取活动名称、时间、地点
- 推送机器人问你：「发现新活动 XXX，是否报名添加提醒？」
- 你回复「是」 → 自动添加到日历，设置活动前提醒

### 2. 智能待办抓取（新增）

**自动扫描飞书群聊/私聊对话**：
- 识别类似「我们周五下午三点开个会」「这个需求下周二要做完」「下周一 10:00 讨论一下」
- 自动提取 **待办内容 + 时间**
- 主动问你：「发现待办：XXX 在 XXXX，是否添加到日程？」
- 你回复「是」 → 自动创建日程，设置提醒

帮你不错过和同事讨论确定的工作事项，不用手动再记一遍。

## 🚀 快速开始

### 1. 环境变量配置

```bash
# 飞书配置
export FEISHU_APP_TOKEN=your_bitable_app_token
export FEISHU_TIME_TABLE_ID=your_time_table_id
export FEISHU_EVENT_TABLE_ID=your_event_table_id
export FEISHU_WEBHOOK=your_robot_webhook
export FEISHU_CALENDAR_ID=your_feishu_calendar_id  # 可选，不填用默认
```

### 2. 定时任务（推荐）

编辑 crontab: `crontab -e`

添加：

```cron
# 每晚 21:00 采集今日数据，发送提醒
0 21 * * * cd /path/to/lark-timeflow && ./scripts/collect-and-report.sh

# 每小时扫描一次群聊找新活动
0 * * * * cd /path/to/lark-timeflow && ./scripts/fetch-events.sh
```

### 3. 分类配置

编辑 `references/app-categories.md` 配置 App → 分类映射，和 time-mirror 用法一样。

## 📋 功能说明

### 每日采集 - `scripts/collect-and-report.sh`

完成：
1. 采集今日 App 数据
2. 聚合整理时间块
3. 写入飞书多维表格
4. 写入 Apple 日历
5. 飞书机器人发送今日时间线，提醒补全空白

### 活动抓取 - `scripts/fetch-events.sh`

完成：
1. 拉取最近 24 小时你加入的各个群的消息
2. 自动识别活动消息（关键词匹配）
3. 提取活动名称/时间/地点
4. 飞书机器人问你是否添加提醒
5. 你回复「是」→ 创建飞书日程 + 设置提醒

### 每周总结 - 自动触发

每周日采集完数据后，自动：
1. 统计本周每日分类时长
2. 生成周报统计
3. 飞书机器人发送周报
4. 对比上周变化，给出趋势

## 🎯 设计原则 - 少即是多，基座设计

> 大多数工具最后都被放弃了，因为太复杂，每天要花十分钟操作，坚持不下来。

lark-timeflow 坚持：

1. ✅ **全自动采集** - 不用你手动记，系统已经记好了
2. ✅ **主动提醒** - 机器人找你，不用你打开 App

6. ✅ **基座定位** - 核心功能做好，保持简洁，可与其他技能配合扩展更多能力

## 🔒 安全边界

### 禁止操作
- ❌ 不会把你的时间数据上传到第三方
- ❌ 不会修改你的系统配置除了你手动添加的 crontab
- ❌ 不会存储你的飞书凭证，只使用环境变量

### 需要确认
- ⚠️ 配置飞书 webhook 需要你提供 token，确认你理解信息安全风险
- ⚠️ 扫描群聊会读取最近消息，全程本地处理，不上传
